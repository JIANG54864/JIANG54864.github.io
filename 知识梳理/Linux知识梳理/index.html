输出重定向中，> 的意思是覆盖, >>的意思是追加

硬链接（ Hard Link）
原文件名和链接文件名都指向相同的物理位址。 (i-node number )
如果删除硬链接文件的源文件，硬链接文件仍然存在，而且保留了原有的内容
硬链接不能跨越文件系统
符号链接（ Symbolic Link）
它的内容是它所链接的文件的路径名
由于存放的是源文件的路径名，因而如果删除符号链接文件的源文件，符号链接文件将无法找到原有资料。
能够跨越文件系统

在移动文件时不需要被移动文件的权限，但需其所在目录具有写权限。
在目录下增删文件、子目录时要有目录的写权限（touch,rm,mkdir,rmdir）
用ls列目录要有目录的读权限
进入目录或将该目录作路径分量时(cd)要有目录的执行权限，故要使用任一个文件，必须有该文件及找到该文件的路径上所有目录分量的执行权限。
仅当要打开一个文件时，即执行涉及到文件内容的操作时，才需要文件的许可。

三、简答下列问题。

1、解释read命令的功能，举例说明其用法。

read命令是从标准输入读入变量值，例如：
read var1 var2
hello world（注：var1为hello，var2为world）
read var1 var2
hello world 0.0（注：var1为hello，var2为world！0.0）
read var1 var2
hello（注：var1为hello，var2为空)

2、如果当前目录下只有一个文件abc123，解释echo abc*和echo xyz=abc*命令的输出。

“echo abc*”命令会输出abc123，“echo xyz=abc*”命令会输出xyz=abc*。
出现这种结果的原因为，“*”既可以作为通配符，也可以作为字符。由于目录下的文件只有abc123，那么在“echo abc”时，*作为通配符，匹配的结果就是abc123。而由于目录下没有以xyz=abc开头的文件，所以在“echo xyz=abc”时，是作为一个字符串进行输出的。*
